# /sales:create-narrative -- Transform product docs into a compelling sales narrative

Takes technical documentation, product specs, or feature descriptions and produces a value-focused sales story.

---

## Process

### Step 1: Gather Input

Accept any of:
- Product documentation
- Feature specifications
- Technical descriptions
- Marketing copy (to improve)
- URL to product page

### Step 2: Extract the Narrative Arc

Frame the analysis around these questions:
- What **PROBLEM** does this solve?
- What's the **TRANSFORMATION** (before → after)?
- Why should the customer **CARE**?
- What **FEELING** does success produce?

Work through the input and produce an 8-24 point story explanation:
1. State the pain the customer feels today
2. Show how the current situation is broken or costly
3. Introduce the product as the change agent
4. Describe the transformation concretely (before → after)
5. Anchor to business outcomes, not features
6. Make it personal — whose problem is this, exactly?

### Step 3: Refine for Sales

Transform the story into sales-ready narrative:

1. **Lead with pain** — What problem are they facing?
2. **Introduce transformation** — What changes with this product?
3. **Prove value** — Concrete benefits, not features
4. **Create urgency** — Why now? Cost of waiting?
5. **End with outcome** — What does success look like?

### Step 4: Generate Talking Points

Extract 3-5 key talking points:
- Each is a standalone value statement
- Memorable and quotable
- Tied to specific customer outcomes

### Step 5: Write Elevator Pitch

Write a 2-3 sentence elevator pitch:
- Natural and conversational
- Something a salesperson would actually say
- Captures core value in 30 seconds

---

## Output Format

```markdown
# Sales Narrative: [Product/Feature Name]

## The Story

[8-24 point narrative, first person, conversational]

## Key Talking Points

1. [First value statement]
2. [Second value statement]
3. [Third value statement]

## Elevator Pitch

"[2-3 sentence script]"

## Pain Point Summary

**Problem:** [One sentence describing the pain]
**Solution:** [One sentence describing the transformation]
**Outcome:** [One sentence describing success]
```

---

## Quality Checklist

- [ ] Focuses on **VALUE** — not features
- [ ] **Customer-centric** — their problem, their outcome
- [ ] **Conversational** — sounds natural when spoken
- [ ] **Specific** — not generic marketing speak
- [ ] **Actionable** — sales team can use immediately
