# /art:d3 -- D3.js Interactive Dashboards

## Purpose

Creates D3.js interactive data visualizations -- code-generated HTML dashboards, NOT AI-generated images. This workflow writes a complete self-contained HTML file with embedded D3.js. The output is an HTML file you open in a browser, not a PNG. Supports bar charts, line charts, scatter plots, force-directed graphs, sankey diagrams, heatmaps, and more.

**This is fundamentally different from other /art commands.** It produces CODE, not images. There is no Generate.ts invocation.

## Output to Downloads First

ALL GENERATED FILES GO TO ~/Downloads/ FIRST -- NEVER directly to project directories.

## When to Use

- "Create a D3 chart for..."
- "Make an interactive visualization of..."
- "Build a dashboard showing..."
- "Visualize this data interactively"
- TELOS consulting dashboards
- Blog post data visualizations
- Network diagrams (architecture, relationships)
- Interactive reports and presentations
- Any data that benefits from hover/click/zoom interactivity

**This is NOT for:**
- Static diagrams (use /art:map or other workflows)
- Editorial illustrations (use /art:comic, etc.)
- Simple infographics (use /art:stat)

## Supported Chart Types

### Charts and Graphs
- **Bar Charts** -- comparisons, rankings, distributions
- **Line Charts** -- trends over time, performance metrics
- **Scatter Plots** -- correlations, clusters, outliers
- **Area Charts** -- cumulative values, stacked comparisons
- **Pie/Donut Charts** -- proportions, percentages

### Network and Relationships
- **Force-Directed Graphs** -- project dependencies, team relationships
- **Tree Diagrams** -- hierarchies, organizational structures
- **Chord Diagrams** -- entity relationships, data flow
- **Sankey Diagrams** -- flow visualization, process mapping

### Advanced
- **Heatmaps** -- intensity, density, correlation matrices
- **Geographic Maps** -- location data, regional analysis
- **Timeline Visualizations** -- project milestones, historical data
- **Custom Dashboards** -- multi-chart compositions

## Color Palette (PAI Standard)

All D3 visualizations MUST use this palette:

```
Primary:
  Deep Purple: #4A148C   (brand accent, primary data series)
  Deep Teal:   #00796B   (secondary accent, secondary data series)
  Charcoal:    #2D2D2D   (text, lines, axes)
  Black:       #000000   (borders, strong emphasis)

Background:
  Sepia:       #EAE9DF   (page background)
  White:       #FFFFFF    (chart area background)

Typography:
  System fonts: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto
  Axis labels: 12px
  Titles: 14-16px
```

For multi-series data, extend with: `d3.schemeCategory10` or `d3.schemeSet3`, but override the first two colors with Purple and Teal.

## Workflow Steps (ALL MANDATORY)

### Step 1: Understand the Data

1. **What data are you visualizing?** Get the actual data or define the data structure.
2. **What insight should the viewer take away?** The visualization must have a point.
3. **What interactivity adds value?** Tooltips, zoom, filtering, click-to-drill-down.

Output:
```
DATA: [Description of data, structure, key fields]
INSIGHT: [What the viewer should understand]
CHART TYPE: [bar / line / scatter / force / sankey / etc.]
INTERACTIVITY: [tooltips / zoom / filter / transitions]
```

### Step 2: Select Chart Type and Plan Layout

Choose from the supported types above. For dashboards with multiple charts, plan the grid layout:

```
LAYOUT:
- Single chart: Full width
- Two charts: Side by side or stacked
- Dashboard: Grid (2x2, 3x1, etc.)

DIMENSIONS:
- Width: 800-1200px (responsive)
- Height: 400-600px per chart
- Margins: top 20, right 30, bottom 40, left 50
```

### Step 3: Write the D3.js Code

Create a complete self-contained HTML file. The file must:

1. **Load D3.js from CDN** -- `<script src="https://d3js.org/d3.v7.min.js"></script>`
2. **Embed data inline** -- no external data files
3. **Apply PAI color palette** -- purple, teal, charcoal, sepia background
4. **Include interactivity** -- tooltips at minimum, zoom/pan where appropriate
5. **Be responsive** -- handle window resize with ResizeObserver
6. **Include error handling** -- validate data, handle empty states
7. **Add accessibility** -- ARIA labels, keyboard navigation

Standard code structure:

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>[Chart Title]</title>
  <script src="https://d3js.org/d3.v7.min.js"></script>
  <style>
    body {
      font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
      background: #EAE9DF;
      margin: 0;
      padding: 20px;
      color: #2D2D2D;
    }
    .chart-container {
      background: #FFFFFF;
      border-radius: 8px;
      padding: 20px;
      max-width: 1000px;
      margin: 0 auto;
      box-shadow: 0 2px 8px rgba(0,0,0,0.1);
    }
    h1 { color: #4A148C; font-size: 18px; margin-bottom: 16px; }
    .tooltip {
      position: absolute;
      background: #2D2D2D;
      color: #FFFFFF;
      padding: 8px 12px;
      border-radius: 4px;
      font-size: 12px;
      pointer-events: none;
      opacity: 0;
      transition: opacity 0.2s;
    }
  </style>
</head>
<body>
  <div class="chart-container">
    <h1>[Chart Title]</h1>
    <svg id="chart"></svg>
  </div>
  <div class="tooltip" id="tooltip"></div>
  <script>
    // Data
    const data = [ /* inline data here */ ];

    // Dimensions
    const width = 800, height = 400;
    const margin = { top: 20, right: 30, bottom: 40, left: 50 };
    const innerWidth = width - margin.left - margin.right;
    const innerHeight = height - margin.top - margin.bottom;

    // SVG setup
    const svg = d3.select('#chart')
      .attr('width', width)
      .attr('height', height)
      .attr('role', 'img')
      .attr('aria-label', '[Description]');

    const g = svg.append('g')
      .attr('transform', `translate(${margin.left},${margin.top})`);

    // Scales, axes, data bindings, interactivity...
    // Use #4A148C for primary, #00796B for secondary
  </script>
</body>
</html>
```

### Step 4: Add Interactive Features

**Tooltips (always include):**
```javascript
const tooltip = d3.select('#tooltip');

function showTooltip(event, d) {
  tooltip.transition().duration(200).style('opacity', 0.9);
  tooltip.html(`Value: ${d.value}`)
    .style('left', (event.pageX + 10) + 'px')
    .style('top', (event.pageY - 28) + 'px');
}

function hideTooltip() {
  tooltip.transition().duration(500).style('opacity', 0);
}
```

**Transitions (for entrance animations):**
```javascript
bars.transition()
  .duration(750)
  .delay((d, i) => i * 50)
  .attr('height', d => yScale(d.value))
  .ease(d3.easeCubicOut);
```

**Zoom and Pan (for dense visualizations):**
```javascript
const zoom = d3.zoom()
  .scaleExtent([0.5, 5])
  .on('zoom', (event) => g.attr('transform', event.transform));
svg.call(zoom);
```

### Step 6: Validate

**Must Have:**
- [ ] Clean, professional appearance
- [ ] PAI color palette applied (purple, teal, charcoal, sepia background)
- [ ] Interactive features working (tooltips at minimum)
- [ ] Responsive to container size
- [ ] Accessible (ARIA labels, keyboard navigation)
- [ ] Data validation in place
- [ ] Error handling for edge cases
- [ ] Self-contained (no external data files)
- [ ] D3.js loaded from CDN

**Must NOT Have:**
- [ ] Generic color schemes (must use PAI palette)
- [ ] Static-only when interactivity makes sense
- [ ] Missing axis labels or legends
- [ ] Overflow or cropped elements
- [ ] External data dependencies
- [ ] Broken interactivity

**Performance Notes:**
- Under 1000 elements: SVG (default, optimal)
- 1000-10000 elements: Consider canvas rendering
- Over 10000 elements: Implement aggregation or virtual scrolling
